# AgentVis 版本更新检测发布流程

本文档描述当前首版更新检测链路：应用和官网读取发布清单，发现新版本后引导用户手动下载。自动签名安装留给后续接入 Tauri updater。

## 发布清单

发布清单样例位于 `website/releases/latest.json`。Cloudflare Worker 或 Workers Assets 需要对外暴露同结构 JSON，推荐路径：

```text
https://<your-domain>/releases/latest.json
```

桌面端默认读取腾讯云 COS 上的：

```text
https://agentvis-download-1318040347.cos.ap-beijing.myqcloud.com/releases/latest.json
```

如果生产环境希望改读 Cloudflare Worker，在构建前设置：

```powershell
$env:VITE_AGENTVIS_RELEASE_MANIFEST_URL = "https://<your-domain>/releases/latest.json"
```

清单里 `platforms.windows-x86_64.url` 指向腾讯云 COS 安装包，`fallbackUrl` 指向备用网盘。Worker 只返回小 JSON，不代理安装包。

## 发版步骤

1. 同步版本号：`package.json`、`src-tauri/tauri.conf.json`、`src-tauri/Cargo.toml`。
2. 构建安装包：`npm run tauri build`。
3. 上传安装包到 COS，例如：

```text
https://agentvis-download-1318040347.cos.ap-beijing.myqcloud.com/releases/0.3.3/AgentVis_0.3.3_x64-setup.exe
```

4. 更新 `website/releases/latest.json` 中的 `version`、`releasedAt`、`notes`、`url`、`fallbackUrl`、`sizeBytes`、`sha256`。
5. 部署website/官网/Worker，并将releases/latest.json 上传到 COS，配置跨域访问规则（见下方），让 `/releases/latest.json` 返回最新清单。
6. 验证官网按钮会更新到新下载地址，桌面端“设置 > 通用 > 版本更新”能检测到新版本。

## 后续自动更新预留

后续接入 `tauri-plugin-updater` 时，可以继续复用当前清单结构，补充 `signature` 字段并在 Worker 端按 Tauri 动态更新接口返回 `version`、`url`、`signature`。无更新时返回 `204 No Content`。

## 配置跨域访问规则
### 第一步：进入 COS 控制台
1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)。
2. 在搜索栏输入 **对象存储 (COS)** 并进入。
3. 在左侧菜单点击 **存储桶列表**，找到你存放那个文件的 **Bucket（存储桶）**。

### 第二步：配置跨域访问规则 (CORS)
1. 点击进入该存储桶后，在左侧导航栏找到 **安全管理** -> **跨域访问设置 (CORS)**。
2. 点击 **添加规则**。

### 第三步：填写参数（对照你图片里的内容）
请按照以下对应关系填写：

*   **来源 Origin**：填写 `*` （代表允许所有网站访问。*如果你知道具体的域名，建议写具体的域名以增强安全性，例如 ：
```
http://tauri.localhost
tauri://localhost
https://agentvis.muulor.workers.dev
```
*   **操作 Methods**：勾选 **GET** 和 **HEAD**。
*   **允许 Headers**：填写 `*`。
*   **Expose-Headers**：依次添加以下三行：
    *   `ETag`
    *   `Content-Length`
    *   `Content-Type`
*   **超时 Max-Age**：填写 `600`。

### 第四步：保存并测试
1. 点击页面下方的 **确认/保存**。
